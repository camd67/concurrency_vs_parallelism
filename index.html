A visual comparison of concurrency and parallelism.

In computer science the words concurrency and parallelism are often used synonymously. This is wrong. These are two very different ideas and we're here to set you straight.

Keep scrolling. Using some typical examples, we'll show you the difference between concurrency and parallelism.

A little background.

From roughly 1980-2005, desktop computers got exponentially faster at running sequential programs. But no one knows how to continue this advancement due to various technological limitations. So, we started putting multiple processors on the same chip.

Concurrency explained.

Concurrency helps with resource allocation. It correctly and efficiently manages access to shared resources. It allows two or more tasks to start, run, and complete in overlapping time periods. This doesn't mean they'll ever be running at the same instant.

Parallelism explained.

With parallelism you are using extra resources to solve a problem faster. With a multicore processor tasks are able to run at the same time.
